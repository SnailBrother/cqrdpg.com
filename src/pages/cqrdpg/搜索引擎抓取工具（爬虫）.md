# React 打包后生成的是一个单页应用，它的 HTML 文件里几乎没有任何实际内容

## 搜索引擎的爬虫通常只会读取 HTML，而不去执行里面的 JS。所以，它看到的就是一个几乎空白的页面，自然也就不会收录你的网站
安装：在你的项目根目录，打开命令行运行：

bash
npm install --save-dev react-snap
修改 package.json：找到 scripts 部分，把 build 命令改成：

  "scripts": {
   
    "build": "react-scripts build",
   
  },


json
"scripts": {
  "build": "react-scripts build && react-snap"
}
重新打包并上传：

再次运行 npm run build。

你会发现 build 文件夹里，每个路由（比如 /about）都生成了一个对应的 .html 文件，里面已经填好了内容。

最后，把新的 build 文件夹里的所有内容，重新上传到你的 Nginx 服务器上就可以了。

这样操作后，爬虫再来访问时，就能直接读取到 HTML 里的完整内容，和普通用户看到的一样。

方案二：确认 Nginx 配置无误
你可以简单检查一下 Nginx 配置，确保路径指对了。

打开 Nginx 配置文件（一般在 /etc/nginx/sites-available/你的域名 或 /etc/nginx/nginx.conf），关键的 root 路径要指向你上传的 build 文件夹内部：

nginx
server {
  listen 80;
  server_name 你的域名.com;  # 换成你的域名

  root /var/www/你的项目文件夹/build;  # 重点：路径要指向 build 文件夹内部，而不是外层
  index index.html;

  location / {
    try_files $uri /index.html;  # 这句很重要，让 React 路由能正常工作
  }
}
改完后，别忘了用 nginx -t 测试配置，没问题的话就 systemctl reload nginx 重载一下。

最后，主动提交给搜索引擎
内容问题解决后，你可以主动“通知”搜索引擎来抓取。

生成 sitemap.xml：可以装一个 sitemap 或 react-router-sitemap 这类工具来生成网站地图，或者找个在线生成器也可以。

提交给站长工具：去 Google Search Console 或 百度搜索资源平台，验证你的网站所有权，然后把 sitemap.xml 提交上去，就能加速收录了。

按这个流程操作一遍，你的网站大概率就能被搜到啦
